3 A library to perform generic seralization and deserialization of QObjects.
4 
5 With this small library, you are able to serialize any QObject or Q_GADGET class to JSON and back. This is done with help of Qt's meta system.
6 
7 # Features {#qtjsonserializer_readme_label_0}
8 - Serialize QObjects, Q_GADGETS, and lists of both to JSON, in a generic matter
9 - ... and of course deserialize JSON back as well
10 - De/serialize any QVariant - as long as it contains only basic types or one of the above
11  - Works even with QJsonValue/Array/Object as propteries
12 - Serializes Q_PROPERTY elements
13 - Enum de/serialization as integer or as string
14 - Deserialization: Additional JSON-values will be stored as dynamic properties for QObjects
15 - Fully Unit-Tested
16 - Thread-Safe
17 - Easily extensible

42 # Usage {#qtjsonserializer_readme_label_2}
43 The serializer is provided as a Qt module. Thus, all you have to do is add the module, and then, in your project, add `QT += jsonserializer` to your `.pro` file!

45 ## Example {#qtjsonserializer_readme_label_3}
46 Both serialization and desertialization are rather simple. Create an object, and then use the serializer as follows:
47 
48 The following is an example for a serializable object. *Note:* The usage of `MEMBER` Properties is not required, and simply done to make this example more readable.
49 ```cpp
50 class TestObject : public QObject
51 {
52  Q_OBJECT
53 
54  Q_PROPERTY(QString stringProperty MEMBER stringProperty)
55  Q_PROPERTY(QList<int> simpeList MEMBER simpeList)
56  Q_PROPERTY(TestObject* childObject MEMBER childObject)
57 
58 public:
59  Q_INVOKABLE TestObject(QObject *parent = nullptr);
60 
61  QString stringProperty;
62  QList<int> simpeList;
63  TestObject* childObject;
64 }
65 ```
66 
67 You can serialize (and deserialize) the object with:
68 ```cpp
69 auto serializer = new QJsonSerializer(this);
70 
71 try {
72  //serialize
73  auto object = new TestObject();
74  object->stringProperty = "test";
75  object->simpeList = {1, 2, 3};
76  object->childObject = new TestObject(object);
77  auto json = serializer->serialize(object);
78  qDebug() << json;
79  delete object;
80 
81  //deserialize
82  object = serializer->deserialize<TestObject>(json);//optional: specify the parent
83  qDebug() << object->stringProperty
84  << object->simpeList
85  << object->childObject;
86  delete object;
87 } catch(QJsonSerializerException &e) {
88  qDebug() << e.what();
89 }
90 ```
91 
92 For the serialization, the created json would look like this:
93 ```json
94 {
95  "stringProperty": "test",
96  "simpeList": [1, 2, 3],
97  "childObject": {
98  "stringProperty": "",
99  "simpeList": [],
100  "childObject": null
101  }
102 }
103 ```

105 ## Important Usage Hints {#qtjsonserializer_readme_label_4}
106 In order for the serializer to properly work, there are a few things you have to know and do:
107 
108 1. Only Q_PROPERTY properties will be serialized, and of those only properties that are marked to be stored (see [The Property System](https://doc.qt.io/qt-5/properties.html#requirements-for-declaring-properties), STORED attribute)
109 2. For deserialization of QObjects, they need an invokable constructor, that takes only a parent: `Q_INVOKABLE MyClass(QObject*);`
110 3. Only properties with the following types can be serialized:
111  - `QObject*` and deriving classes
112  - Classes/structs marked with `Q_GADGET` (as value types only!)
113  - `QList`, of any type that is serializable as well
114  - Simple types, that are supported by QJsonValue (See [QJsonValue::fromVariant](https://doc.qt.io/qt-5/qjsonvalue.html#fromVariant) and [QJsonValue::toVariant](https://doc.qt.io/qt-5/qjsonvalue.html#toVariant))
115  - `Q_ENUM` and `Q_FLAG` types, as integer or as string
116  - The string de/serialization of Q_ENUM and Q_FLAG types only works if used as a Q_PROPERTY. Integer will always work.
117  - `QJson...` types
118  - Any type you add yourself by extending the serializer
119 4. While simple list types (i.e. `QList<int>`) are supported out of the box, for custom types (like `QList<TestObject*>`) you will have to register converters from and to `QVariantList`
120  - This can be done by using [`QJsonSerializer::registerListConverters<TestObject*>()`](src/qjsonserializer.h#L27)
121 5. By default, the `objectName` property of QObjects is not serialized (See [keepObjectName](src/qjsonserializer.h#L20))
122 6. By default, the JSON `null` can only be converted to QObjects/G_GADGETs. For other types the conversion fails (See [allowDefaultNull](src/qjsonserializer.h#L19))
