12 # Features {#qtjsonserializer_readme_label_0}
13 - Serialize QObjects, Q_GADGETS, lists, maps, etc. to JSON, in a generic matter
14 - ... and of course deserialize JSON back as well
15 - De/serialize any QVariant - as long as it contains only basic types or one of the above
16  - Works even with QJsonValue/Array/Object as properties
17 - Serializes Q_PROPERTY elements
18 - Enum de/serialization as integer or as string
19 - Deserialization: Additional JSON-values will be stored as dynamic properties for QObjects
20 - Supports polymorphism
21 - Fully Unit-Tested
22 - Thread-Safe
23 - Easily extensible

56 # Usage {#qtjsonserializer_readme_label_2}
57 The serializer is provided as a Qt module. Thus, all you have to do is install the module, and then, in your project, add `QT += jsonserializer` to your `.pro` file!
58 
59 ## Example {#qtjsonserializer_readme_label_3}
60 Both serialization and desertialization are rather simple. Create an object, and then use the serializer as follows:
61 
62 The following is an example for a serializable object. *Note:* The usage of `MEMBER` Properties is not required, and simply done to make this example more readable.
63 ```cpp
64 class TestObject : public QObject
65 {
66  Q_OBJECT
67 
68  Q_PROPERTY(QString stringProperty MEMBER stringProperty)
69  Q_PROPERTY(QList<int> simpeList MEMBER simpeList)
70  Q_PROPERTY(TestObject* childObject MEMBER childObject)
71 
72 public:
73  Q_INVOKABLE TestObject(QObject *parent = nullptr);
74 
75  QString stringProperty;
76  QList<int> simpeList;
77  TestObject* childObject;
78 }
79 ```
80 
81 You can serialize (and deserialize) the object with:
82 ```cpp
83 auto serializer = new QJsonSerializer(this);
84 
85 try {
86  //serialize
87  auto object = new TestObject();
88  object->stringProperty = "test";
89  object->simpeList = {1, 2, 3};
90  object->childObject = new TestObject(object);
91  auto json = serializer->serialize(object);
92  qDebug() << json;
93  delete object;
94 
95  //deserialize
96  object = serializer->deserialize<TestObject>(json);//optional: specify the parent
97  qDebug() << object->stringProperty
98  << object->simpeList
99  << object->childObject;
100  delete object;
101 } catch(QJsonSerializerException &e) {
102  qDebug() << e.what();
103 }
104 ```
105 
106 For the serialization, the created json would look like this:
107 ```json
108 {
109  "stringProperty": "test",
110  "simpeList": [1, 2, 3],
111  "childObject": {
112  "stringProperty": "",
113  "simpeList": [],
114  "childObject": null
115  }
116 }
117 ```

119 ## Important Usage Hints {#qtjsonserializer_readme_label_4}
120 In order for the serializer to properly work, there are a few things you have to know and do:
121 
122 1. Only Q_PROPERTY properties will be serialized, and of those only properties that are marked to be stored (see [The Property System](https://doc.qt.io/qt-5/properties.html#requirements-for-declaring-properties), STORED attribute)
123 2. For deserialization of QObjects, they need an invokable constructor, that takes only a parent: `Q_INVOKABLE MyClass(QObject*);`
124 3. Only the following types can be serialized:
125  - `QObject*` and deriving classes
126  - Classes/structs marked with `Q_GADGET` (as value types only!)
127  - `QList`, of any type that is serializable as well
128  - `QMap<QString, T>`, with T beeing any type that is serializable as well (string as key type is required)
129  - Simple types, that are supported by QJsonValue (See [QJsonValue::fromVariant](https://doc.qt.io/qt-5/qjsonvalue.html#fromVariant) and [QJsonValue::toVariant](https://doc.qt.io/qt-5/qjsonvalue.html#toVariant))
130  - `Q_ENUM` and `Q_FLAG` types, as integer or as string
131  - The string de/serialization of Q_ENUM and Q_FLAG types only works if used as a Q_PROPERTY. Integer will always work.
132  - `QJson...` types
133  - `QPair<T, T>`, of any types that are serializable as well
134  - Standard QtCore types (e.g. QPoint, QSize, QVersionNumber, ...)
135  - Any type you add yourself by extending the serializer
136 4. While simple types (i.e. `QList<int>`) are supported out of the box, for custom types (like `QList<TestObject*>`) you will have to register converters. This goes for
137  - QList and QMap: use `QJsonSerializer::registerAllConverters<T>()`
138  - QList only: use `QJsonSerializer::registerListConverters<T>()`
139  - QMap only: use `QJsonSerializer::registerMapConverters<T>()`
140  - QPair: use `QJsonSerializer::registerPairConverters<T1, T2>()`
141  - QSharedPointer/QPointer: use `QJsonSerializer::registerPointerConverters<T>()`
142 5. Polymorphic QObjects are supported. This is done by the serializer via adding a special `@@class` json property. To make a class polymorphic you can:
143  - Add `Q_CLASSINFO("polymorphic", "true")` to its definition
144  - Globally force polyphormism (See QJsonSerializer::polymorphing in the doc)
145  - Set a dynamic property: `setProperty("__qt_json_serializer_polymorphic", true);`
146 6. By default, the `objectName` property of QObjects is not serialized (See [keepObjectName](src/qjsonserializer.h#L20))
147 7. By default, the JSON `null` can only be converted to QObjects. For other types the conversion fails (See [allowDefaultNull](src/qjsonserializer.h#L19))

149 ## Support for alternative Containers {#qtjsonserializer_readme_label_5}
150 Right now, only `QList` and `QMap` ar supported as containers. The reason is, that adding containers requires the registration of converters. Supporting all containers would explode the generated binary, which is why I only support the most common ones.
151 
152 If you need the other containers, you have 2 options:
153 
154 1. Implement a custom `QJsonTypeConverter` (You will still have to register all the converters).
155 2. Create "converter" properties that are used for serialization only. This is the more simple version, but needs to be done for every occurance of that container, and adds some overhead.
156 
157 The following example shows how to do that to use `QVector` in code, but serialize as `QList`:
158 ```
159 struct MyGadget {
160  Q_GADGET
161 
162  Q_PROPERTY(QList<int> myIntVector READ getMyIntList WRITE setMyIntList) //normal property name, as this one appears in json
163  //Important: Add "STORED false":
164  Q_PROPERTY(QVector<int> myIntVectorInternal READ getMyIntVector WRITE setMyIntVector STORED false) //will not be serialized
165 
166 public:
167  QVector<int> getMyIntVector() const;
168  void setMyIntVector(const QVector<int> &vector) ;
169 
170 private:
171  QList<int> getMyIntList() const {
172  return getMyIntVector().toList();
173  }
174  void setMyIntList(const QList<int> &list) {
175  setMyIntVector(QVector<int>::fromList(list));
176  }
177 };
178 ```
